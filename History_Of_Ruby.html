When was Ruby created, and by who?

Ruby was first created in 1995 by Japanese programmer Yukihiro "Matz" Matsumoto. He created this language that balances "fuctional programming with imperative programming. This means that he wanted to make a object-oriented programmaning language, thus became Ruby.

The General History of the language

  • Ruby 0.95 (1995)

    Matz released the first version of Ruby in Japan, December of 1995.

  • Ruby 1.0, 1.1, & 1.2 (1996-1998)

    One year after the release of 0.95, Ruby 1.0 was released. In August of 1997, Ruby 1.1 was released, but the first stable version of Ruby was released in December of 1998, which was Ruby 1.2.

  • Ruby-Talk (1998)

    In 1998, Matz created a Ruby homepage in English & made the first language Ruby mailing list Ruby-Talk. This is where ruby was beginning to spread beyond Japan.

  • The Object-oriented scripting Language Ruby (1999)

    In October of 1999, Both Matsumoto & Keiju Ishitsuka worked together to make the first book on the programming language. By this point, the popularity of Ruby is starting to really grow and get popular in Japan, as well as it spreading rapidly to outside countries.

  • Ruby, Programming Ruby (2001)

    In the year of 2001, the FIRST English book on Ruby was published. This allowed people outside of Japan, specifically English speakers, to learn more about this new language at the time.

  • Ruby 1.8 & RubyGems (2003-2004)

    In 2003, Ruby 1.8 is released, which included new features like Duck Typing, Fully Qualified Names, Native YAML Support, WEBrick, StringIO, open-uri, Pretty Printer for Hash#inspect, and much more. In 2004, RubyGems was released. It served as an assit for Ruby.

  • Ruby on Rails (2005)

    In 2005, Ruby on Rails was made. This had a new feature that is the "Model-View-Controller", which works on "convention over configuration", which is an efficent way to make web applications. This gained so much attention to were people loved it, and made Ruby more popular than ever.

  • Ruby 1.8.6 & 1.8.7...and Apple?! (2007-2008)

    In March of 2007, version 1.8.6 of Ruby was released, which is an improvement over the previous versions of Ruby. In May of 2008, 1.8.7 of Ruby was released, where the same thing can be said about it as 1.8.6, an improvment over the previous versions of Ruby. Apple took a interest with this language, and began implementing their devices with it, like the iPhone!

  • Ruby 1.9, 1.9.2, and 1.9.3 (2007-2011)

    In December of 2007, the development version of Ruby was released, which was 1.9. This version then got a stable release on 2011 as Ruby 1.9.3. The new changes it brought were, speed improvements, newly implemented methods, new has syntax, RubyGems, and more.

  • Ruby 2.0.0 (2013)

    In February of 2013, Ruby 2.0.0 was released. This version brung more improvements to the language, such as speed improvemnts, and optimized garbage collection, keyword arguments and more.

  • Ruby 2.1.0, 2.1.1, and 2.1.2 (2013-2014)

    In December of 2013, Ruby 2.1.0 was released. This version brung some small changes, but the biggest One were semantic versioning. In February of 2014, Ruby 2.1.1 was released. This version had speed improvements while also fixing some bugs. Three months later, Ruby 2.1.2 was released in May of 2014, though it like 2.1.1, it also fixes bugs that the previous versions of Ruby had before.

What types of problems is the language used for?

As you seen and learned above, the Ruby language was a way to professionally build websites efficiently, while also being compatible with making programs.

What is the current state of the language?

As the time of writing this, Ruby 3.2.3 is the current version of this programming language, which released on January of 2024, where it is still used to make websites and programs!
